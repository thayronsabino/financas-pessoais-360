# Roadmap — MordomIA

> **Versão atual:** v2.1.0 (Maio/2026)  
> **Última atualização:** 2026-05-08  
> **Repositório:** https://github.com/thayronsabino/mordomia

Este documento descreve a visão de longo prazo do **MordomIA** — o sistema operacional financeiro fundamentado em princípios de mordomia cristã. Não é apenas uma lista de funcionalidades futuras; é o **caminho consciente** de transformação do pacote em uma plataforma viva.

---

## Filosofia do Roadmap

```
PRINCÍPIO 1 — Inteligência antes de automação
  Cada nova versão amplia a capacidade consultiva ANTES de adicionar 
  ferramentas operacionais. Risco: virar "app de finanças comum".

PRINCÍPIO 2 — Profundidade antes de amplitude
  Melhor 7 skills excelentes do que 30 medianas. Melhor 5 playbooks 
  premium do que 50 tutoriais.

PRINCÍPIO 3 — Mordomia bíblica como núcleo, sempre
  Toda evolução respeita os Princípios Inegociáveis. Nada vira "geral" 
  às custas da identidade cristã.

PRINCÍPIO 4 — Acolhimento antes de tecnicismo
  Cada feature passa pelo teste pastoral: "isso ajuda a alma de quem 
  está sofrendo financeiramente?"

PRINCÍPIO 5 — Acessibilidade antes de sofisticação
  Antes de adicionar funcionalidade complexa, garantir que o pacote 
  funciona para o usuário em SOBREVIVÊNCIA com baixa familiaridade.
```

## 🛡️ Princípios que NÃO Mudam

Estes são **invariantes** do MordomIA. Qualquer versão futura, em qualquer cronograma, **deve respeitar**:

```
1. DÍZIMO 10% É PISO ABSOLUTO
   Não há versão futura onde dízimo abaixo de 10% será sugerido.
   Em qualquer estado financeiro, em qualquer crise, em qualquer país.

2. ACOLHIMENTO ANTES DE NÚMEROS
   Toda primeira interação começa com escuta empática.
   Nenhuma "otimização" pode pular essa fase.

3. ESTADO REAL, NÃO ESTADO DESEJADO
   Sistema NUNCA dá orientação de Expansão para usuário em Sobrevivência.
   "Onde a pessoa está" sempre define "para onde pode ir hoje".

4. SISTEMA RECONHECE LIMITES
   Quando o problema transcende finanças, encaminha para apoio profissional.
   Não tenta substituir terapeuta, pastor, advogado, médico.

5. MORDOMIA OPERACIONAL + ESPIRITUAL
   As duas dimensões juntas, sempre.
   Tecnologia financeira sem alma é app comum.
   Espiritualidade sem técnica é boa intenção desorganizada.

6. GRATUITO PARA USO LOCAL
   O pacote em arquivos locais será sempre gratuito.
   SaaS futuro será conveniência opcional, não dependência.

7. CÓDIGO ABERTO (Apache 2.0)
   Versões futuras mantêm licença permissiva.
   Outros podem usar, modificar, distribuir, comercializar.
   Única condição: referenciar fonte original.

8. PRINCÍPIOS BÍBLICOS COMO NÚCLEO
   Mesmo internacionalização não dilui o núcleo cristão.
   Pacote pode ser usado por não-cristãos, mas não vira "secular" para 
   agradar mercado.
```

## 📊 Critérios de Release

Para que uma versão seja lançada, ela precisa atender:

### Critérios Técnicos

- [ ] Todos os arquivos com frontmatter atualizado (versão, data)
- [ ] `REFERENCIAS-BRASIL-2026.md` validado contra fontes oficiais
- [ ] Cross-references entre arquivos verificados
- [ ] Sem links quebrados ou paths incorretos
- [ ] Glossário atualizado com novos termos (se houver)

### Critérios de Qualidade

- [ ] Pelo menos 1 caso de uso completo documentado para cada nova feature
- [ ] Revisão pastoral (alguém com formação em aconselhamento bíblico)
- [ ] Revisão técnica (alguém com formação financeira: CFP, contador, economista)
- [ ] Teste com pelo menos 3 usuários reais antes do release público

### Critérios de Princípio

- [ ] Mudanças passam pelo teste dos 8 invariantes acima
- [ ] Nenhuma alteração reduz a dimensão pastoral
- [ ] Nenhuma alteração aumenta barreira para usuário em SOBREVIVÊNCIA

## 🤝 Como Contribuir

O MordomIA é open source (Apache 2.0). Contribuições são bem-vindas:

### Tipos de contribuição

```
1. CORREÇÃO DE TAXAS / DADOS:
   Pull Request atualizando REFERENCIAS-BRASIL-2026.md
   Sempre citando fonte oficial

2. NOVO PLAYBOOK:
   Caso brasileiro/internacional não coberto
   Seguir estrutura dos playbooks existentes
   Validar com pelo menos um conselheiro pastoral + um financeiro

3. NOVO FRAMEWORK:
   Decisão financeira específica e frequente
   Engine de decisão claro (IF/THEN)
   Aplicação prática + casos de uso

4. SCRIPTS DE SIMULAÇÃO:
   Python/JavaScript com testes
   Documentação clara de inputs/outputs
   Aderência ao padrão do REFERENCIAS-BRASIL-2026.md

5. TRADUÇÃO:
   Skills, playbooks ou frameworks em outros idiomas
   Adaptação cultural respeitando princípios bíblicos universais

6. CASOS DE USO REAIS (anonimizados):
   Transcrições de jornadas reais (com permissão)
   Mostram o pacote em ação
   Inspiram outros usuários
```

### Processo de contribuição

```
1. Fork: github.com/thayronsabino/mordomia
2. Branch: feature/[descricao] ou fix/[descricao]
3. Commit: mensagens claras, em português ou inglês
4. PR: descrição do problema + solução + testes
5. Review: mantenedores avaliam alinhamento com princípios
6. Merge: após aprovação técnica + pastoral
```

### O que NÃO aceitamos

```
❌ Conteúdo que sugere dízimo abaixo de 10%
❌ Endividamento como solução para qualquer problema
❌ Esquemas de "ficar rico rápido"
❌ Pirâmides financeiras / esquemas Ponzi
❌ Investimentos em criptomoedas voláteis sem ressalvas pesadas
❌ Conteúdo que pula acolhimento e vai direto para técnica
❌ Linguagem moralista ou de julgamento
❌ Substitutos para terapia, advocacia ou medicina
```

## 🎯 Métricas de Sucesso (Visão de Longo Prazo)

Não medimos sucesso por:
- Número de downloads
- Estrelas no GitHub
- "Engajamento" (tempo de uso é frequentemente sintoma de mau design)

Medimos sucesso por:
- **Histórias de transformação** documentadas e voluntariamente compartilhadas
- **Saída do estado SOBREVIVÊNCIA** — quantas pessoas migraram para ORGANIZAÇÃO em 6 meses
- **Aumento de generosidade sustentável** — pessoas crescendo em dízimo + ofertas
- **Redução de endividamento por usuário** ao longo do tempo
- **Adoção em comunidades cristãs** — igrejas usando para discipulado financeiro
- **Reconhecimento pastoral** — pastores recomendando o sistema

## 🙏 Visão Final

Em 5-10 anos, o MordomIA quer:

```
▸ Servir 100.000+ famílias cristãs no Brasil
▸ Estar traduzido em pelo menos 3 idiomas
▸ Ser referência em mordomia financeira biblicamente fundamentada
▸ Ter parcerias com igrejas, seminários e ministérios
▸ Permanecer 100% gratuito para uso pessoal
▸ Ser usado em discipulado financeiro estruturado
▸ Inspirar outras ferramentas com mesma filosofia
```

Mas, acima de tudo:

```
▸ Continuar sendo um sistema que reconhece que dinheiro mexe com alma.
▸ Que ferramentas servem pessoas, não o contrário.
▸ Que mordomia bíblica é mais que técnica — é uma forma de ser fiel.
▸ Que finanças organizadas são infraestrutura para uma vida com propósito,
  nunca o destino final.
```
